# Teacher Portal — User Guide & Configuration

## Overview

The **Teacher Portal** is the dedicated interface for teachers at a Ugandan primary school to access their dashboard, manage grades, and reset passwords. All routes are namespaced under `/teacher/` and require teacher authentication (no access to non-teacher accounts).

## Employee ID Generation

Employee IDs are generated automatically during teacher registration using the format:

```
SCHOOLPREFIX-YY-NNNN
```

**Format Breakdown:**
- **SCHOOLPREFIX** — School code (e.g., `"KPS"` for Kampala Primary School)
- **YY** — Two-digit year of registration (e.g., `24` for 2024)
- **NNNN** — Four-digit sequential counter (e.g., `0001`, `0002`, etc.)

**Example:** `KPS-24-0001`, `KPS-24-0002`

The ID is generated by the `generate_employee_id()` utility function in `SchoolNowMgt/registration/utils.py` and cannot be manually edited.

## Password Rules

Teachers must follow these **4 password strength rules** when registering and resetting passwords:

1. **Minimum 8 characters** — Password must be at least 8 characters long.
2. **At least 1 uppercase letter** — Must contain A–Z.
3. **At least 1 digit** — Must contain 0–9.
4. **At least 1 special character** — Must contain one of: `! @ # $ % ^ & * ( ) _ + -`

**Example valid password:** `Kampala2024!`

**Example invalid password:** `password` (no uppercase, no digit, no special char)

## Curriculum Support

### Uganda Curriculum (Primary)

- **Classes:** P1 through P7
- **Terms:** 1, 2, 3
- **Score Range:** 0–100
- **Grading Scale:** A, B, C, D, E, F

Teachers enter raw scores (0–100) which are automatically converted to letter grades.

**Score-to-Grade Mapping:**
- A: 80–100
- B: 70–79
- C: 60–69
- D: 50–59
- E: 40–49
- F: 0–39

### International Curriculum

- **IGCSE (Cambridge):**
  - Grades: A*, A, B, C, D, E, F, G, U
  - Semesters: 1, 2
- **IB (International Baccalaureate):**
  - Grades: 7, 6, 5, 4, 3, 2, 1
  - Semesters: 1, 2

Teachers select the **subject type** (IGCSE or IB) and enter grades directly without conversion.

## First Login Experience

When a teacher logs in for the first time:

1. A **welcome banner** is displayed on the dashboard (auto-dismisses after one view or after X seconds).
2. The banner shows the teacher's **employee ID** (e.g., `KPS-24-0001`) prominently.
3. A prompt may appear to complete their profile (qualifications, profile photo) if not already done.
4. The dashboard displays:
   - **Today's schedule** (classes, start times)
   - **My students** (count and classes assigned)
   - **Attendance overview** (present/absent today in my classes)
   - **At-risk students** (retention alerts for my classes)
   - **Recent grades** (grades entered by this teacher)

The first-login flag is determined by checking:
- `request.user.last_login is None` (never logged in)
- `request.user.date_joined.date() == today` (registered today)

## Adding Subjects to a Teacher's Timetable

**Important:** Subjects and timetables are **NOT edited in the teacher portal**. They are managed by school administrators via the **Django Admin panel** only.

### Steps for Admin:

1. Log in to `/admin/` with a staff/superuser account.
2. Navigate to **Timetable** in the left menu.
3. Click **Add Timetable**.
4. Fill in:
   - **Teacher** — Select the teacher
   - **Subject** — Select the subject
   - **Class Grade** — Select the class
   - **Day of Week** — e.g., "monday"
   - **Start Time** & **End Time** — e.g., 08:00–09:00
5. Save.

The teacher will now see this class in their dashboard and can enter grades for it.

## Environment Variables & Email Configuration

### For Password Reset Emails

To enable password reset functionality, configure an **email backend** in `schoolmgmt_project/settings.py`:

#### Development (Console Output):
```python
EMAIL_BACKEND = 'django.core.mail.backends.console.EmailBackend'
```

Emails are printed to the console (useful for testing).

#### Production (Gmail SMTP Example):
```python
EMAIL_BACKEND = 'django.core.mail.backends.smtp.EmailBackend'
EMAIL_HOST = 'smtp.gmail.com'
EMAIL_PORT = 587
EMAIL_USE_TLS = True
EMAIL_HOST_USER = 'your-email@gmail.com'           # Environment variable recommended
EMAIL_HOST_PASSWORD = 'your-app-password'          # Environment variable recommended
DEFAULT_FROM_EMAIL = 'School Name <noreply@yourschool.com>'
PASSWORD_RESET_TIMEOUT = 86400  # 24 hours
```

### Environment Variables Recommended:
```
EMAIL_HOST_USER=your-email@gmail.com
EMAIL_HOST_PASSWORD=your-app-password-or-token
DEFAULT_FROM_EMAIL=School Name <noreply@yourschool.com>
```

Use a package like `python-decouple` or `python-dotenv` to load these from a `.env` file:
```python
from decouple import config
EMAIL_HOST_USER = config('EMAIL_HOST_USER')
EMAIL_HOST_PASSWORD = config('EMAIL_HOST_PASSWORD')
```

## Troubleshooting

### "Page Not Found" at /teacher/login/
- Ensure `'teacher'` is in `INSTALLED_APPS` in `settings.py`.
- Verify `schoolmgmt_project/urls.py` includes `path('teacher/', include('teacher.urls', namespace='teacher'))`.

### "Permission Denied" or "Redirect to Login"
- Ensure your user account has `role='teacher'` in the `CustomUser` model.
- Teachers must register via `/teacher/register/` or have the role set manually via Django admin.

### Password Reset Emails Not Sending
- Check `settings.py` — `EMAIL_BACKEND` may be set to `console` (development only).
- Verify SMTP credentials (Gmail, SendGrid, etc.) if using production email.
- Check Django logs for SMTP errors.

### Grades Not Showing in Dashboard
- Verify the teacher has assigned classes via the **Django admin** (`Timetable` model).
- Ensure grades were entered via `/teacher/grades/uganda/` or `/teacher/grades/international/`.

## Security Notes

1. **Teacher Role Validation:** All teacher portal views check `request.user.role == 'teacher'` to prevent non-teacher access.
2. **Login Required:** Except for registration, login, logout, and password reset flows — all views require `@login_required`.
3. **POST-Only Logout:** The logout view only accepts POST requests to prevent CSRF attacks.
4. **Safe Redirects:** Login success redirects are validated to prevent open redirect attacks.
5. **Email Filter:** Password reset forms filter results by `role='teacher'` to prevent accidental resets to parent/admin accounts.
